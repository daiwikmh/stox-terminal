# Read-Only Mode

Read-only mode lets an AI agent access live market data without any secrets, wallet, or 1Password. No write operations are possible.

## Setup

1. Open the Stoxy terminal → right sidebar → **Agent** tab → **OpenClaw**.
2. Click **Connect OpenClaw**.
3. Copy the generated system prompt.
4. Paste it into your AI model.
5. Tell the AI: "Use **Option A — Read-only mode**."

The AI will confirm it understands the mode and proceed to Step 1 (discover skills).

## Authentication

All read-only endpoints require the session token in a header:

```
X-Agent-Token: <your-token>
```

The token is a UUID generated by `POST /api/token/generate` and included in the system prompt.

## Available endpoints

### GET /api/skills

Returns the list of capabilities the agent can call.

```bash
curl http://localhost:8090/api/skills?token=YOUR_TOKEN
```

### GET /api/context

Returns what the user is currently viewing in the browser.

```bash
curl "http://localhost:8090/api/context?token=YOUR_TOKEN"
```

**Response:**
```json
{
  "network": "TESTNET",
  "account_id": "GABC...",
  "active_pair": "XLM/USDC",
  "recent_trades": [...],
  "open_offers": [...]
}
```

### GET /api/prices

Returns all mark prices tracked by the matching engine.

```bash
curl http://localhost:8090/api/prices
```

**Response:**
```json
{
  "XLM/USDC": 0.112345,
  "NVDA/USD": 142.80
}
```

### GET /api/bridge/pairs

Lists all available trading pairs.

```bash
curl http://localhost:8090/api/bridge/pairs \
  -H "X-Agent-Token: YOUR_TOKEN"
```

### GET /api/bridge/orderbook

Live order book for a pair.

```bash
curl "http://localhost:8090/api/bridge/orderbook?symbol=XLM/USDC" \
  -H "X-Agent-Token: YOUR_TOKEN"
```

**Response:**
```json
{
  "bids": [{"price": "0.1123", "amount": "5000"}],
  "asks": [{"price": "0.1124", "amount": "3200"}]
}
```

### GET /api/bridge/price

Mid-price for a single pair.

```bash
curl "http://localhost:8090/api/bridge/price?symbol=XLM/USDC" \
  -H "X-Agent-Token: YOUR_TOKEN"
```

### GET /api/bridge/offers

Open SDEX offers for an account.

```bash
curl "http://localhost:8090/api/bridge/offers?account=GABC..." \
  -H "X-Agent-Token: YOUR_TOKEN"
```

### GET /api/bridge/trades

Recent trade history for an account.

```bash
curl "http://localhost:8090/api/bridge/trades?account=GABC...&limit=20" \
  -H "X-Agent-Token: YOUR_TOKEN"
```

### GET /api/bridge/trustline

Check whether an account has a trustline for an asset.

```bash
curl "http://localhost:8090/api/bridge/trustline?account=GABC...&asset=USDC" \
  -H "X-Agent-Token: YOUR_TOKEN"
```

## Network selection

All bridge endpoints respect the `X-Stellar-Network` header:

```
X-Stellar-Network: TESTNET
X-Stellar-Network: MAINNET
```

Omit the header to default to TESTNET.

## What read-only mode cannot do

- Place, fill, or cancel orders
- Open or close leveraged positions
- Submit signed transactions
- Access admin endpoints

For trading capabilities, see [Trading Mode](trading-mode.md).
